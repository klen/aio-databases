# AGENTS.md — aio-databases

## Repository overview

`aio-databases` is a Python 3.10+ library that provides async database access for SQLite,
PostgreSQL, and MySQL.
It supports both `asyncio` and `trio`, and wraps drivers like `aiosqlite`, `asyncpg`, `aiopg`,
`aiomysql`, `trio_mysql`, and `aioodbc`.

### Project structure

| Path | Description |
|------|-------------|
| `aio_databases/` | Main source code |
| `aio_databases/database.py` | Core `Database` class and connection management |
| `aio_databases/backends/` | Driver-specific backends (e.g., `_asyncpg.py`, `_aiosqlite.py`, `common.py`) |
| `aio_databases/record.py` | Row record abstraction |
| `aio_databases/types.py` | Type definitions |
| `aio_databases/url.py` | URL parsing and redaction |
| `tests/` | Pytest test suite |
| `tests/conftest.py` | Shared fixtures (backend parametrization, DB URLs, pypika-orm models) |
| `pyproject.toml` | Project metadata, tool configs (ruff, pytest, pyrefly, bumpversion) |
| `uv.lock` | Locked dependency tree (managed by `uv`) |

### Technology stack

- **Package manager**: `uv` (do not use pip/poetry directly)
- **Build backend**: `uv_build`
- **Linter/formatter**: `ruff` (line length 100, target Python 3.10)
- **Type checker**: `pyrefly`
- **Testing**: `pytest` with `pytest-aio[trio]`
- **Pre-commit**: conventional commits, ruff, pyrefly, uv-lock check

## High-risk commands

Avoid these unless explicitly instructed:

| Command | Risk |
|---------|------|
| `make release` / `make patch` / `make minor` | Bumps version, creates git tags, merges `develop` into `master`, and pushes to origin. |
| `uv publish` | Publishes to PyPI. |
| `docker start postgres mysql` | Assumes local Docker containers exist; will fail if they don't. |
| `git push --atomic` (in Makefile) | Part of the release flow; do not run manually. |
| Manual edits to `uv.lock` | Regenerated by `uv sync` / `uv lock`. |
| Manual version bumps in `pyproject.toml` | Use `bump-my-version` or the Makefile targets instead. |

## Safe validation

Run these to check your changes without side effects:

```bash
# Sync dependencies (safe)
uv sync

# Lint and format
uv run ruff check
uv run ruff format

# Type check
uv run pyrefly check

# Run only SQLite/backend-agnostic tests (no Docker required)
uv run pytest tests/test_base.py tests/test_sqlite.py tests/test_queries.py tests/test_connections.py tests/test_transactions.py

# Full test suite (requires Docker containers: postgres, mysql)
make test
```

## Editing guidelines

- **Python version**: Target 3.10+.
  Use `from __future__ import annotations` and `|` union syntax.
- **Line length**: 100 characters (enforced by ruff).
- **Imports**: Follow the existing style; group standard library, third-party, and local imports.
- **Type annotations**: The project uses `pyrefly`.
  Keep type coverage accurate; check with `uv run pyrefly check`.
- **Backend changes**: If modifying `aio_databases/backends/common.py`,
  note that it is the base for all backends.
  Changes may affect `_asyncpg.py`, `_aiomysql.py`, `_aiopg.py`, `_aiosqlite.py`, `_aioodbc.py`,
  and `_trio_mysql.py`.
- **Tests**: The test suite in `tests/conftest.py` parametrizes over multiple backends.
  Prefer adding tests that exercise `db` fixture behavior rather than backend internals
  when possible.

## Repo-specific notes

- **Default branch for development**: `develop`.
  `master` is the stable branch.
  Do not open PRs against `master` unless explicitly asked.
- **Conventional commits**: Use types `feat`, `fix`, `perf`, `refactor`, `style`, `test`, `build`,
  `ops`, `docs`, `merge`.
  Config is in `.git-commits.yaml`.
- **Trio support** is limited: only `trio-mysql` backend supports `trio`;
  other backends are `asyncio`-only.
- **Docker required for full tests**: PostgreSQL and MySQL tests expect running containers.
  The SQLite tests are safe to run anywhere.
- **Optional dependencies**: Drivers are optional extras (`asyncpg`, `aiomysql`, `aiosqlite`, etc.).
  The core library has no heavy runtime dependencies.

## Good defaults for agents

- Always run `uv run ruff check` and `uv run pyrefly check` before considering a task done.
- Run SQLite-only tests (`tests/test_base.py`, `tests/test_sqlite.py`)
  when you can't verify Docker services.
- Keep changes focused to one concern (backend, connection logic, queries, etc.).
- Do not commit `uv.lock` unless you intentionally changed dependencies in `pyproject.toml`.
